这份清单专为 2026 年美国全栈实习面试 定制，旨在通过技术选型展示你的 Senior 架构审美 和对 Modern Web (T3 Stack) 趋势的掌控力。

# 前端架构

## 核心框架 (Core Framework)

Next.js 15+ (App Router)

- 竞品/替代方案: Vite (React SPA), Remix

### 选型理由 & 面试核心卖点

- 架构深度：展示你对 React Server Components (RSC)、Streaming (流式渲染) 和 Suspense 边界的深度理解。
- 性能：仪表盘首屏需要快速加载，SSR (服务端渲染) 是刚需。
- 路由能力：利用 Parallel Routes 并行加载不同的图表组件，利用 Intercepting Routes 实现无缝的模态框交互。

## 编程语言 (Language)

TypeScript (Strict Mode)

- 竞品/替代方案: JavaScript (ES6+)

### 选型理由 & 面试核心卖点

- 工程健壮性：必须开启 `strict: true`。配合 Zod 实现运行时校验。
- Senior 信号：展示你如何使用 Generics (泛型) 和 Utility Types 处理复杂的 API 数据结构，而非仅仅使用 `any`。

## 状态管理 (Client State)

Zustand

- 竞品/替代方案: Redux Toolkit, Context API

### 选型理由 & 面试核心卖点

- 去样板代码：Redux 在美国新项目中已显得笨重。Zustand 极简且符合 Hooks 心智模型。
- 适用场景：仅用于管理纯 UI 状态（如 Sidebar 折叠、用户偏好设置），避免过度设计。

## 服务端状态与缓存 (Async State)

TanStack Query (React Query v5)

- 竞品/替代方案: useEffect + fetch, SWR

### 选型理由 & 面试核心卖点

- 分离关注点：明确区分 "Client State" 和 "Server State"。
- Dashboard 杀手锏：利用 `staleTime`、`refetchInterval` (轮询) 和 `Window Focus Refetching` 轻松实现宏观经济数据的实时动态刷新，无需手写复杂的 WebSocket 或轮询逻辑。

## UI 样式与组件系统 (Styling & UI)

Tailwind CSS (v4) + shadcn/ui

- 竞品/替代方案: Material UI (MUI), Ant Design, Styled-components

### 选型理由 & 面试核心卖点

- Headless UI 哲学：`shadcn/ui` (基于 Radix UI) 不是安装包，而是复制代码。展示你懂 Accessibility (无障碍) 和 Composition (组合式) 设计。
- 现代标准：Tailwind 是目前美国业界绝对的样式标准。展示你对 Utility-first CSS 的熟练度和对 Runtime CSS overhead 的厌恶。

## 数据可视化 (Data Visualization)

Tremor (或 Recharts)

- 竞品/替代方案: D3.js, Chart.js, Highcharts

### 选型理由 & 面试核心卖点

- SaaS 风格：Tremor 是专为 Dashboard 设计的 React 库，风格极其 Modern，开箱即用。
- 效率与深度的平衡：如果面试官问底层，你可以提及 Recharts 的 SVG 原理；如果问效率，Tremor 是构建 B2B 仪表盘的最优解。

# 后端架构

## 后端运行时与 API (Runtime)

Next.js API Routes + Server Actions

- 竞品/替代方案: Nest.js, Express, Fastify

### 选型理由 & 面试核心卖点

- BFF 模式：对于 Dashboard 项目，Next.js 足以作为 BFF (Backend for Frontend)。
- 前沿特性：重点展示 Server Actions，这允许你在前端组件中直接调用后端函数（RPC 风格），这是 2024-2026 React 生态最大的范式转移。

## 全栈通信协议 (Communication)

tRPC

- 竞品/替代方案: REST API, GraphQL

### 选型理由 & 面试核心卖点

- 端到端类型安全：这是全栈 TS 项目的皇冠明珠。后端修改数据库 Schema 或 API 输出，前端代码编译时直接报错。
- 开发体验 (DX)：展示你对开发效率和代码质量的极致追求，面试时演示 "Code completion across the stack" 会非常震撼。

## 数据库与 ORM (Database Layer)

PostgreSQL + Drizzle ORM

- 竞品/替代方案: MongoDB, Prisma, TypeORM

### 选型理由 & 面试核心卖点

- 轻量化趋势：虽然 Prisma 是标准，但 Drizzle 因其 "SQL-like" 和 "Zero-runtime overhead" 正在成为 2026 年的新宠。
- Serverless 友好：Drizzle 启动极快，非常适合部署在 Edge 环境。选它能体现你对新技术趋势的敏锐嗅觉。

## 身份认证 (Authentication)

Clerk

- 竞品/替代方案: NextAuth.js (Auth.js), Firebase Auth

### 选型理由 & 面试核心卖点

- 集成能力：不要手写 Auth！Clerk 提供了目前最好的 React 组件和中间件支持。
- 关注业务：展示你懂得利用高质量 SaaS 工具来减少非核心业务的开发成本。

# 工程化与 DevOps

## 包管理与构建工具 (Build Tools)

pnpm + Turborepo

- 竞品/替代方案: npm, yarn, Lerna

### 选型理由 & 面试核心卖点

- Monorepo 思维：即使是单人项目，使用 Turborepo 也能展示你对大型项目架构（Workspace、任务缓存）的理解。
- 效率：pnpm 的磁盘空间节省和安装速度是加分项。

## 代码规范与工具链 (Linting)

Biome.js

- 竞品/替代方案: ESLint + Prettier

### 选型理由 & 面试核心卖点

- Rust 浪潮：Biome 是用 Rust 编写的，旨在统一 Formatter 和 Linter。使用它能暗示你关注 Web Infrastructure 领域的 Rust 化趋势（如 Turbopack, Rolldown）。

## 测试框架 (Testing)

Vitest (单元测试) + Playwright (E2E)

- 竞品/替代方案: Jest, Cypress

### 选型理由 & 面试核心卖点

- 生态统一：Vitest 与 Vite/Next 配置兼容，速度极快。
- E2E 标准：Playwright 是微软出品，运行速度和稳定性优于 Cypress。针对核心的“金融数据展示”写测试，体现工程素养。

## 部署与 CI/CD (Deployment)

Vercel + GitHub Actions

- 竞品/替代方案: AWS EC2, Docker

### 选型理由 & 面试核心卖点

- 预览环境：利用 Vercel 的 Preview Deployments 进行代码审查流程。
- 自动化：配置 GitHub Actions 在 Push 时自动运行类型检查 (`tsc`) 和测试，确保持续集成的质量。
